Methodology & Data Processing

This section details the technical framework used to process the MobileCN dataset, ensuring high fidelity and privacy compliance across all visualization layers.

1. Data Collection Strategy

The dataset is aggregated from mobile base stations across major Chinese cities. We utilize a distributed collection system that captures:

  • Active User Counts (Anonymized)
  • Traffic Throughput (Upload/Download in TB)
  • Timestamp markers (15-minute intervals)
  • Geospatial coordinates (WGS-84)

2. Data Cleaning Pipeline

Raw data often contains noise and missing values. We apply the following steps:

> Step 1: Outlier Removal (Z-Score > 3)
> Step 2: Spatial Interpolation (Kriging Method)
> Step 3: Privacy Masking (K-Anonymity)
> Step 4: Normalization (Min-Max Scaling)

3. Visualization Engine

The dashboard is powered by Apache ECharts and customized WebGL layers to render high-density scatter plots. The rendering logic optimizes performance by clustering data points at low zoom levels and revealing granular details at high zoom levels.

4. Algorithm Reference

For the urban planning module, we implement a flow prediction model based on LSTM (Long Short-Term Memory) networks to forecast traffic peaks.
